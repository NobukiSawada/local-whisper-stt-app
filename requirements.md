# 指示

あなたは優秀なPythonエンジニアです。以下の要件定義に基づき、Windowsローカル環境で動作する音声文字起こしアプリの開発をサポートしてください。
今回はプロジェクトの第一歩として、全体像を把握した上で「Step 1: PC内音声の取得モジュール」のコードを作成してください。

# 1. 開発背景・目的

日々のオンライン会議（Zoom等）や動画などのPC内音声をテキスト化し、記録やデータ分析に活用したいと考えています。しかし、機密性の高い情報を含む可能性があるため、外部の有料APIにデータを送信したくありません。
そのため、Hugging Faceの軽量かつ高精度な音声認識モデルを活用し、完全ローカル環境で完結するセキュアなアプリケーションを開発することにしました。

# 2. システム・非機能要件

- OS: Windows環境専用
- 実行環境: 完全ローカル（モデルの初回ダウンロード時を除く）
- コスト: 外部APIを利用しない（完全無料）
- パフォーマンス: 実用的な処理速度と、高精度な日本語認識の両立

# 3. 機能要件

- 音声入力: PC内の音声（ループバック/スピーカー出力）のみを取得。マイク入力は不要。
- 文字起こし: 取得した音声をリアルタイム（または数秒ごとのチャンク）で文字起こしし、画面にテキストを表示する。
- 保存機能: 画面上のテキストを `.txt` 形式で任意のローカルディレクトリに保存できる。
- UI: 最低限のボタン（開始・停止・保存）とテキストエリアのみの簡潔な画面。

# 4. 技術スタック（想定）

- 言語: Python 3.x
- 音声取得: `soundcard` (Windowsループバックの取得用)
- 推論モデル: `faster-whisper` (軽量化・高速化されたWhisper)
- GUI: `customtkinter` (または簡潔に実装できるもの)

# 5. ディレクトリ構成（予定）

win-loopback-transcriber/
├── main.py # GUIとメインループ（アプリのエントリポイント）
├── audio_capture.py # 音声取得処理モジュール
├── transcriber.py # 文字起こし処理モジュール
├── requirements.txt
└── README.md
